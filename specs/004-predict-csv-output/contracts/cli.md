# CLI Contract: Predict CSV Output

## `predict` Command

### Command

```bash
npm run cli -- predict --input <input-directory>
```

### Existing Output

The command continues to emit one JSONL `prediction` record per processed image to the existing command output stream.

### New CSV Artifact

Each run creates or replaces this file before image processing begins:

```text
<input-directory>/p.csv
```

The file header is always written first, even when the input directory has no supported images.

### CSV Header

Generated `p.csv` files must use this exact ordered header:

```csv
imageId,time,hand,systolic,diastolic,pulse,status,confidence,uncertainFields,provider,model,rawNotes
```

### Row Semantics

- One row is written as each image is processed.
- Rows are written for complete, partial, unreadable, and error predictions.
- Null or missing values are written as empty cells.
- `uncertainFields` is written as a JSON array string in one cell, then CSV-escaped when necessary.
- Values containing commas, quotes, carriage returns, or line feeds are escaped using standard CSV quoting.

### Example

```csv
imageId,time,hand,systolic,diastolic,pulse,status,confidence,uncertainFields,provider,model,rawNotes
img001,2026-05-19 06:05:20,left,121,75,75,complete,0.98,[],openai,gpt-5.4-mini,
img002,,right,127,72,69,partial,0.82,"[""time""]",openai,gpt-5.4-mini,No supported embedded timestamp metadata found
img003,,,,,,error,,"[""time""]",openai,gpt-5.4-mini,provider failed
```

## `eval` Command

### Command

```bash
npm run cli -- eval --input <input-directory> --csv <input-directory>/p.csv
```

### Semantics

`eval` treats the supplied CSV as reference data. When a generated `p.csv` is passed through `--csv`, eval still runs fresh predictions for images in `--input` and compares those fresh predictions against the core reading columns from `p.csv`.

### Accepted CSV Columns

Eval requires these core columns:

```csv
imageId,time,hand,systolic,diastolic,pulse
```

Eval ignores any additional columns, including:

```csv
status,confidence,uncertainFields,provider,model,rawNotes
```

### Rejection Rules

Eval rejects the CSV with a clear error when:

- any required core column is missing
- duplicate `imageId` values are present
- `hand` contains an unsupported non-empty value
- numeric fields contain malformed non-empty text

### Compatibility Guarantee

Every `p.csv` generated by `predict` is accepted by `eval --csv` without manual editing when used as reference data for the same image directory.
